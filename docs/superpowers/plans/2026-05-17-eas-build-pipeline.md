# EAS Build Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Configure EAS builds for both Amana apps (Agent + Principal) producing signed AABs (Android) and IPAs (iOS) via EAS cloud builds, ready for store submission.

**Architecture:** Each app gets its own `eas.json` with three profiles (development, preview, production). EAS managed credentials for iOS; Android uses Play App Signing (EAS generates upload key, Google manages app signing key). Version bumped to 1.0.0, `android.versionCode` = 100, `ios.buildNumber` = "1". Native permission strings added to `app.json` for camera, location, and NFC.

**Tech Stack:** EAS CLI (`eas-cli`), Expo SDK 51, App Store Connect, Google Play Console

**Spec:** `docs/superpowers/specs/2026-05-17-app-store-submission-design.md`

### Task 1 — Prerequisite: verify Android Play App Signing + install EAS CLI

**Files:** none

Before touching any code, confirm the Android signing situation. This determines whether EAS can generate a fresh upload key or whether you need to locate the original keystore.

- [ ] **Step 1: Check Play App Signing in Play Console**

1. Go to [Google Play Console](https://play.google.com/console)
2. Open the Amana app that has the existing closed testing build
3. Navigate to: **Release → Setup → App signing**
4. Look for two distinct sections: **"App signing key certificate"** and **"Upload key certificate"**

If both sections are present with separate SHA-1 fingerprints: Play App Signing is enrolled ✅ — proceed.

If only one key is shown with no upload key separation: Play App Signing is NOT enrolled ⚠️. Stop here. The original keystore must be located before any new builds can be uploaded. Check your Expo account dashboard (expo.dev → legacy builds) for a managed keystore, or find the local `.jks` file from the machine that originally built the app.

- [ ] **Step 2: Install EAS CLI globally**

```bash
npm install -g eas-cli
```

Verify installation:

```bash
eas --version
```

Expected output: `eas-cli/x.x.x linux-x64 node-vxx.x.x` (version number will vary, any recent version is fine).

- [ ] **Step 3: Log in to Expo**

```bash
eas login
```

Enter your Expo account email and password when prompted. If you don't have an Expo account, create one at [expo.dev](https://expo.dev) first (free).

Verify login:

```bash
eas whoami
```

Expected: your Expo username printed.

### Task 2 — Register Agent with EAS + create `eas.json`

**Files:**
- Create: `apps/agent/eas.json`
- Modify: `apps/agent/app.json` (EAS init adds `extra.eas.projectId`)

- [ ] **Step 1: Register the Agent project with EAS**

```bash
cd apps/agent
eas init
```

EAS will prompt:

```
? Would you like to create a project for @<your-username>/amana-agent? › Yes
```

Answer **Yes**. EAS adds the following block to `apps/agent/app.json` automatically:

```json
"extra": {
  "eas": {
    "projectId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }
}
```

The UUID is generated by EAS — do not edit it.

- [ ] **Step 2: Create `apps/agent/eas.json`**

```json
{
  "cli": {
    "version": ">= 12.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal",
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "autoIncrement": "buildNumber"
    }
  }
}
```

- [ ] **Step 3: Commit**

```bash
git add apps/agent/eas.json apps/agent/app.json
git commit -m "feat(agent): EAS project registration + eas.json build profiles"
```

### Task 3 — Update Agent `app.json`: version, permissions, plugins

**Files:**
- Modify: `apps/agent/app.json`

This task adds the version fields, native permission strings (camera, location, NFC), NFC entitlement, and plugin declarations needed for a successful store build. The `extra.eas.projectId` added by `eas init` in Task 2 must be preserved.

- [ ] **Step 1: Check Play Console for Agent's current versionCode**

In Play Console, open the Agent app → **Release → Closed testing** → click the existing build → look for **"Version code"** in the build details.

If the Agent app has no existing Play Console entry: use `versionCode: 1`.
If unsure: use `versionCode: 100` — it is always safe to jump ahead.

- [ ] **Step 2: Replace `apps/agent/app.json`**

Write the complete file below, substituting `YOUR_PROJECT_ID` with the UUID that `eas init` added in Task 2 (copy it from the current `app.json` before overwriting):

```json
{
  "expo": {
    "name": "Amana Agent",
    "slug": "amana-agent",
    "version": "1.0.0",
    "orientation": "portrait",
    "userInterfaceStyle": "automatic",
    "ios": {
      "bundleIdentifier": "com.amana.agent",
      "supportsTablet": false,
      "buildNumber": "1",
      "infoPlist": {
        "NSCameraUsageDescription": "Used to scan QR codes and capture receipt photos.",
        "NSLocationWhenInUseUsageDescription": "Used to record the location of transactions.",
        "NFCReaderUsageDescription": "Used to pair with a principal device via NFC tap."
      },
      "entitlements": {
        "com.apple.developer.nfc.readersession.formats": ["TAG"]
      }
    },
    "android": {
      "package": "com.amana.agent",
      "versionCode": 100
    },
    "platforms": ["ios", "android"],
    "plugins": [
      "expo-secure-store",
      [
        "expo-notifications",
        { "color": "#222222" }
      ]
    ],
    "extra": {
      "eas": {
        "projectId": "YOUR_PROJECT_ID"
      }
    }
  }
}
```

- [ ] **Step 3: Verify the projectId is present and correct**

```bash
grep -A3 '"extra"' apps/agent/app.json
```

Expected: the `projectId` UUID is present, not the literal string `YOUR_PROJECT_ID`.

- [ ] **Step 4: Commit**

```bash
git add apps/agent/app.json
git commit -m "feat(agent): app.json — v1.0.0, versionCode 100, NFC + camera + location permissions"
```

### Task 6 — First production build: Agent

**Files:** none (cloud build — no local file changes)

EAS builds run in Expo's cloud infrastructure. This takes ~15–20 minutes per platform. Both iOS and Android run simultaneously.

- [ ] **Step 1: Trigger the Agent production build**

```bash
cd apps/agent
eas build --platform all --profile production
```

**iOS credential prompts (first time only):**

EAS prompts you to authenticate with Apple and set up signing credentials. When asked:
- Enter your Apple ID (`alexanderadegbola@gmail.com`) and password
- For distribution certificate: choose **"Generate a new Apple Distribution Certificate"** (or reuse an existing one if offered)
- For provisioning profile: choose **"Generate a new provisioning profile"**
- For push notifications: choose **"Skip"** or **"Set up later"** — APNs setup is deferred to sub-project 3

EAS stores the generated certificate and profile in its credentials service. You won't need to repeat this for subsequent builds.

**Android credential prompts (first time only):**

When asked whether EAS should manage your Android keystore, answer **Yes**. EAS generates a new upload keystore and stores it securely. Google Play will associate this upload key with your app — future AABs must be signed with the same upload key (EAS handles this automatically).

> **Note on Android notifications:** `expo-notifications` is included as a plugin. EAS cloud builds route Android push notifications through Expo's own Firebase project by default — you do NOT need to provide `google-services.json` for the build to succeed. If you see a Firebase-related build error, re-run `eas build` and accept the default FCM configuration when prompted.

- [ ] **Step 2: Monitor the build**

EAS prints a URL when the build starts:

```
Build details: https://expo.dev/accounts/<username>/projects/amana-agent/builds/<build-id>
```

Open this URL to watch live build logs. Alternatively:

```bash
eas build:list --limit 4
```

Wait until both `android` and `ios` builds show `FINISHED` status. If either shows `ERRORED`, read the build logs at the URL above — most first-build errors are credential or permission issues.

- [ ] **Step 3: Download and smoke-test the Android APK**

Download the APK artifact from the EAS build:

```bash
cd apps/agent
eas build:download --platform android
```

This saves the APK to the current directory. Install it on a connected Android device or running emulator:

```bash
adb install amana-agent-*.apk
```

Expected: App installs, launches to the login screen. If it crashes immediately, check that all `app.json` permissions are present and the bundle installed correctly.

- [ ] **Step 4: Verify iOS IPA is available**

iOS builds cannot be installed directly without TestFlight or a provisioning profile. Confirm the IPA is downloadable from the EAS dashboard — that's sufficient verification for this sub-project.

Go to: `https://expo.dev/accounts/<username>/projects/amana-agent/builds` → click the iOS build → confirm "Download" button is present.

- [ ] **Step 5: Commit build record (no code changes)**

```bash
git tag v0.0-eas-agent-build-1
git push origin v0.0-eas-agent-build-1
```
